# PochiClock - CLAUDE.md

## 概要
Smiley全部門向け勤怠管理Webアプリ。スマホ・PC操作が苦手な人でも使えるシンプルさが最優先。

**技術スタック**: Laravel 12 + Tailwind CSS + MySQL + UUID

## 要件（確定済み）

### 打刻方式（2方式併用）

| 方式 | 対象部門 | フロー | 工数 |
|------|---------|--------|------|
| PIN打刻 | 配食（調理・配達） | キオスクで名前タップ → 4桁PIN | 2ステップ |
| QR打刻 | 美容 | スマホQR → タブレットカメラ | 2ステップ |
- 配食部はスマホ苦手な人が多い → キオスク端末のみで完結するPIN方式
- 美容部は既にQR運用実績あり＋タブレットあり → QR方式継続
- 介護・居宅・タクシー → 事務所出勤時はPCキオスク+PIN、直行直帰時はスマホPWA+PIN+GPS
- **タブレットは美容部にしかない** → 他拠点はPC or スマホで完結する設計
- ICカードは廃止（発行の手間＋紛失リスク）
- PIN忘れ → 管理者がリセットできる機能を実装

### キオスク設定
- 各キオスク端末にデフォルト部署を設定可能
- URL: `/kiosk/{department}` （現状の実装をそのまま活用）
- タブレットのホーム画面に部署URLをブックマーク登録するだけ
- 他部署の人が使う場合は部署切替ボタンで変更可能

### マルチテナント方針

**方式: 行レベル分離（自前Global Scope実装）**

- 全テーブルに `tenant_id` (UUID) カラム追加
- Eloquent Global Scopeで全クエリに `WHERE tenant_id = ?` を自動付与
- `BelongsToTenant` トレイトを全モデルに適用
- パッケージ不使用（Laravel標準機能のみ）
- CIでトレイト未適用モデルを検出（漏れ防止）

**選定理由:**
- 50社×50名=274万行はMySQLで余裕。DB分離は過剰
- 移行コスト最小（10〜15日）、運用負荷最小（1DB管理）
- 将来100社超えたらパーティショニングで拡張可能

**法的対策（勤怠データ5年保存義務）:**
- テナント単位のデータエクスポート機能を標準搭載
- 論理削除 + 保持期限管理（`retention_until` = 解約日+5年）
- `withoutGlobalScope` 使用箇所は全て監査ログに記録

### タスクステータス即時更新（必須）

**作業が進んだら、その場でタスクのnext_actionとstatusを更新する。セッション終了まで待たない。**

以下のいずれかに該当したら即更新:
- 成果物（ファイル・コード・設計書等）を完成した
- git commit & push した
- ブロッカーが解消された（前提機能の完成等）
- フェーズが進んだ（要件定義→実装等）

**next_actionの書き方**: 完了した内容（過去形）ではなく、**次にやるべきこと**を書く。

```bash
# 更新例
python -c "
import urllib.request, json
url = 'https://twinklemark.xsrv.jp/nicolio-api/api.php/tasks?id=eq.{TASK_UUID}'
data = json.dumps({'next_action': '次にやること', 'status': '進行中'}).encode()
req = urllib.request.Request(url, data=data, headers={
    'Authorization': 'Bearer nicolio_secret_2525xsrv',
    'Content-Type': 'application/json'
}, method='PATCH')
urllib.request.urlopen(req)
"
```

### セッション完了時プロトコル（必須）

セッション終了前に**必ず**以下を実行:

1. **git commit & push** — 未コミット変更を残さない
2. **タスク更新** — next_actionが「次のステップ」を指しているか確認。完了したアクションが残っていたら書き換える
3. **git status** — 未コミットがないことを最終確認
